Description Curtis Rueden 2013-09-03 14:14:10 CDT 
As reported by an anonymous IRC user today: with a fully updated Fiji, changing a preference in the Edit > Preferences menu, quitting Fiji, and then opening Fiji again resets the preference back to its default value.

The user wrote: "The problem is that it seems that preferences actually are saved, because when I change anything manually in .imagej/IJ_Prefs.txt it gets reverted back to original when I close Fiji. Or better said back to default. If I change anything manually before opening Fiji it is not reflected."

Bug occurs on both Ubuntu Linux 12.04 64-bit and 13.04 64-bit (and very likely other Linuxes), with both OpenJDK 1.7.0 and Fiji's default shipped version of Java 6.

My (very speculative) guess is that the problem might be related to the ImageJ2 legacy layer options synchronization, which is Johannes's *favorite* part of ImageJ!
