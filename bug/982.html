Comment 1 Mark Hiner 2015-01-02 07:21:17 CST 
Hi Liz,

1) If you run Edit > Options > ImageJ2, are either of the boxes checked? If so, try unchecking these options. See http://imagej.net/ImageJ2 for more information.

If these options are already disabled, would you mind using Help > Upload Sample Image to send us a dataset that is opening more slowly than it used to - so we can test and confirm on our end?

2) I'm less certain what to do about this issue - I personally wasn't able to reproduce the problem. The behavior you mentioned (time to open an image, "Scrolling tool" text) is expected for the actions you took. So just to confirm - if you have the Fiji application selected and move your mouse over the pixels of an image, the status bar remains completely unchanged?

If so I would recommend trying the following, in order:

a) Run Help > Update Fiji again. Your bug report indicates you still have a few files out of date. They do not look like anything that would affect the mouse probe like this, but it can't hurt to update.

b) If that doesn't help, try downloading a fresh Fiji from http://fiji.sc/Downloads

c) If that doesn't work, try running with a newer version of Java (e.g. 1.7): http://imagej.net/FAQ#How_do_I_launch_ImageJ_with_a_different_version_of_Java.3F

d) And if a new Java doesn't change anything, would you mind downloading ImageJ by itself from http://imagej.nih.gov/ij/download.html and seeing if you get coordinates and pixel values on mouse-over?

e) Finally, do you have any other systems to test this behavior on? e.g. Windows or a more recent version of OSX?

I'm sorry I don't have more specific suggestions here, but I hope these steps will help to identify the conditions that are causing this behavior on your system.

Thank you for the report!

Happy new year,
Mark
