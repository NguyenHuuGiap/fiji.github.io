Sorry, for my previous 'back-handed bug report', I hope I am now doing things the proper way. I can reproduce the bug I reported to the FIJI-devel list. Here is a more detailed list of issues I encountered with update sites and the ImageJ updater:

in some cases the updater does not automatically mark for installation all the files that are present in the FIJI update site (this bug seems to be variable as it sometimes works for unknown reasons, here is a non working example)
http://youtu.be/GqOloe3ftwE
here is an example where it kind of worked, both tests were done on the same day with the same up-to-date FIJI installation (in my hands it most often does not work though especially with the http://sites.imagej.net/Aigouy/ site and it most often works with http://srv-gred.u-clermont1.fr/labmirouse/software/FIJI_update_site/)

Another non intuitive feature of the 'manage update sites' window is that the user cannot add a name if he hasn't first entered a URL, as you can see from the video I always make the mistake, probably because it is more intuitive for me to fill columns from left to right and therefore to enter a name first and then only enter the URL. Another annoying thing is that whenever I uncheck the checkbox associated to a FIJI update site, I had previously manually added, it is forever gone from my list when I restart FIJI/relaunch the updater (please always have a look at the last line in the 'manage update sites' window to see what I mean)
http://youtu.be/G57ndGsmm_g

hope this will help making the updater which is already great even greater.

best,

Benoit
