Description Cornel Pop 2014-05-04 11:32:07 CDT 
Running a simple "import sys" in the Macro.ijm tool, with the language set to Python, will cause the Debug window to be flooded by "Waiting for 2 threads to die" messages as soon as the "script" is finished running. No other python code is necessary to trigger the bug, just the import statement. This renders the Debug tool almost useless, since any other debugging message will just flash by, and the only way to see it is to save the debug file and to find and read the relevant output in a text editor. While this bug is annoying, it may also indicate deeper problems with threading. The only way to stop the message flooding is to restart Fiji.
