After some more investigation, it seems that this is linked to the offscreen
rendering - as a workaround one can run Fiji with:

  ./fiji -Dj3d.noOffScreen=true --

I can reproduce a similar crash with the OffScreenTest example in j3d-examples
with Java3D versions 1.5.1, 1.5.2 and current daily snapshots, so I've
reported this upstream:
